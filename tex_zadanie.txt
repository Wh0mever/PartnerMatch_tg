**ТЕХНИЧЕСКОЕ ЗАДАНИЕ

Telegram-бот «Партнёрский Центр Организаций»**

1. Цель проекта

Создать Telegram-бот, который автоматизирует поиск партнёров между организациями, позволяет обмениваться файлами, создавать типовые договоры, проходить верификацию, находить наставников и получать консультации от ресурсного центра.

Бот работает как «знакомства между компаниями» по критериям: каждая компания выставляет что она может дать и что ей нужно. Совпадения дают возможность открыть контакты и вести диалог.

2. Роли пользователей
1) Владелец

• Полный доступ ко всем данным
• Управление администраторами
• Утверждение / блокировка организаций
• Инструменты аналитики

2) Администратор

• Проверка регистраций
• Видеозвонок для проверки реальности организации
• Подтверждение / отклонение заявок
• Блокировка нарушителей
• Добавление курсов и материалов

3) Ресурсный центр

• Консультации по поиску партнёров
• Загрузка информации о конкурсах
• Ответы на вопросы по процессам

4) Наставник

• Пользователь, который помогает другим участникам
• Может принимать заявки от тех, кто ищет наставника

5) Организация

• Основной пользователь
• Ищет партнёров, проходит верификацию, обменивается файлами, создаёт договоры

4. Поиск партнёров (основа бота)
4.1. Лента организаций

Пользователь видит организации, которые:

прошли верификацию

подходят по обороту

совпадают по критериям «могу дать» / «нужно»

4.2. Фильтр по обороту

Перед просмотром пользователь выбирает:

категории оборотов, допустимых для поиска

4.3. Механика лайков

На каждой анкете:

«+» (интересно)

«–» (не интересно)

Если обе стороны нажали «+»:

Открываются контакты обеих сторон

Открывается чат для дальнейшей переписки (через бот)

12. Основные сценарии работы
A. Регистрация

Пошаговые вопросы → отправка → проверка → активация профиля.

B. Поиск партнёров

Лента анкет → +/– → матч → чат.

Форма → PDF → отправка → архив.

D. Наставничество

Заявка → подтверждение → чат.

E. Новости

Создать → отправить → публикация в общей ленте.
